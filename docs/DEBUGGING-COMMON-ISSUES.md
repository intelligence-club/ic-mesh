# Debugging Common Issues - IC Mesh

This guide covers the most frequently encountered issues in IC Mesh and provides step-by-step troubleshooting instructions with specific commands and solutions.

## 🚀 Quick Diagnostic Commands

Before diving into specific issues, run these commands to get a system overview:

```bash\n# System health check\nnode diagnostic-center.js\n\n# Check server status\ncurl -s http://localhost:8333/status | jq .\n\n# View recent logs\ntail -50 mesh.log\n\n# Database quick stats\necho \"SELECT COUNT(*) as total_jobs, \n       SUM(CASE WHEN status='pending' THEN 1 ELSE 0 END) as pending,\n       SUM(CASE WHEN status='completed' THEN 1 ELSE 0 END) as completed\n       FROM jobs;\" | sqlite3 mesh.db\n```\n\n## 🔧 Node Connection Issues\n\n### Problem: \"Node not registering with server\"\n\n**Symptoms:**\n- Node starts but doesn't appear in `/nodes` endpoint\n- No heartbeat messages in logs\n- Jobs not being claimed by node\n\n**Diagnosis Steps:**\n\n1. **Check node configuration:**\n```bash\n# Verify config file exists and is valid JSON\ncat node-config.json | jq .\n\n# Check required fields\njq '.nodeId, .serverUrl, .capabilities' node-config.json\n```\n\n2. **Test network connectivity:**\n```bash\n# Test server reachability\ncurl -v http://localhost:8333/status\n\n# Test from node's perspective (if remote)\ncurl -v http://YOUR_SERVER_IP:8333/status\n```\n\n3. **Check node logs:**\n```bash\n# Look for registration errors\nnode client.js --verbose 2>&1 | grep -i \"register\\|error\\|fail\"\n```\n\n**Common Solutions:**\n\n- **Wrong server URL:** Update `serverUrl` in `node-config.json`\n- **Firewall blocking:** Open port 8333 on server\n- **Invalid nodeId:** Use unique identifier (avoid duplicates)\n- **Missing capabilities:** Add at least one capability to the list\n\n### Problem: \"Node keeps disconnecting\"\n\n**Symptoms:**\n- Node registers but disappears from active list\n- Intermittent job processing\n- Frequent reconnection attempts\n\n**Diagnosis:**\n\n1. **Check heartbeat interval:**\n```bash\n# Monitor heartbeat messages\nnode client.js --debug 2>&1 | grep heartbeat\n```\n\n2. **Check server-side node status:**\n```bash\n# See when node was last seen\necho \"SELECT nodeId, lastHeartbeat, datetime(lastHeartbeat/1000, 'unixepoch') \n       FROM nodes WHERE nodeId LIKE 'YOUR_NODE%';\" | sqlite3 mesh.db\n```\n\n**Solutions:**\n- Reduce heartbeat interval in node config\n- Check network stability\n- Verify system resources aren't constraining the node\n\n## 💼 Job Processing Problems\n\n### Problem: \"Jobs stuck in 'pending' state\"\n\n**Symptoms:**\n- Jobs submitted but never claimed\n- Queue building up\n- No nodes processing work\n\n**Diagnosis:**\n\n1. **Check active nodes:**\n```bash\n# Count active nodes\ncurl -s http://localhost:8333/nodes | jq 'length'\n\n# Check node capabilities\ncurl -s http://localhost:8333/nodes | jq '.[].capabilities'\n```\n\n2. **Check job requirements vs. node capabilities:**\n```bash\n# See pending jobs and their requirements\necho \"SELECT jobId, type, requirements FROM jobs WHERE status='pending';\" | sqlite3 mesh.db\n```\n\n3. **Check for quarantined nodes:**\n```bash\n# Look for quarantined nodes\necho \"SELECT nodeId, quarantined, quarantineReason FROM nodes WHERE quarantined=1;\" | sqlite3 mesh.db\n```\n\n**Solutions:**\n- **No capable nodes:** Start nodes with required capabilities\n- **Quarantined nodes:** Investigate and fix node issues, then unquarantine\n- **Mismatched requirements:** Adjust job requirements or node capabilities\n\n### Problem: \"Jobs failing with handler errors\"\n\n**Symptoms:**\n- Jobs claimed but fail to complete\n- Error messages like \"Handler not found\" or \"Exit 1\"\n- Low success rate\n\n**Diagnosis:**\n\n1. **Check recent job failures:**\n```bash\n# Get recent failed jobs with error messages\necho \"SELECT jobId, type, error_message, datetime(completedAt/1000, 'unixepoch') \n       FROM jobs WHERE status='failed' \n       ORDER BY completedAt DESC LIMIT 10;\" | sqlite3 mesh.db\n```\n\n2. **Test handler availability:**\n```bash\n# On the node, test if handler exists\nls -la handlers/\n\n# Test specific handler\nnode handlers/transcribe.js --test\n```\n\n3. **Check dependencies:**\n```bash\n# For transcribe jobs - check Whisper installation\nwhich whisper\nwhisper --version\n\n# For OCR jobs - check Tesseract\nwhich tesseract\ntesseract --version\n```\n\n**Solutions:**\n- **Missing handlers:** Install required handler files\n- **Missing dependencies:** Install required system dependencies\n- **Permission errors:** Fix file permissions on handlers\n- **Configuration errors:** Verify handler configuration\n\n## 🗄️ Database Issues\n\n### Problem: \"Database locked\" errors\n\n**Symptoms:**\n- \"Database is locked\" error messages\n- Server failing to start\n- Unable to submit jobs\n\n**Diagnosis:**\n\n1. **Check for competing processes:**\n```bash\n# Find processes using the database\nlsof mesh.db\n\n# Check for zombie processes\nps aux | grep -E '(server\\.js|client\\.js|node)' | grep -v grep\n```\n\n2. **Check database integrity:**\n```bash\n# Test database access\necho \".schema\" | sqlite3 mesh.db\n\n# Check for corruption\necho \"PRAGMA integrity_check;\" | sqlite3 mesh.db\n```\n\n**Solutions:**\n```bash\n# Stop all processes using the database\npkill -f server.js\npkill -f client.js\n\n# If corruption detected, restore from backup\ncp mesh.db mesh.db.corrupt\ncp backups/mesh-$(date +%Y%m%d).db mesh.db\n```\n\n### Problem: \"Corrupted timestamp data\"\n\n**Symptoms:**\n- Jobs showing creation dates in the far future\n- Negative \"minutes ago\" values\n- Diagnostic errors about timestamp corruption\n\n**Diagnosis:**\n```bash\n# Find corrupted timestamps\necho \"SELECT COUNT(*) as corrupted_jobs FROM jobs \n       WHERE createdAt > strftime('%s','now') * 1000;\" | sqlite3 mesh.db\n\necho \"SELECT COUNT(*) as corrupted_nodes FROM nodes \n       WHERE lastHeartbeat > strftime('%s','now') * 1000;\" | sqlite3 mesh.db\n```\n\n**Solution:**\n```bash\n# Run timestamp repair utility\nnode fix-node-timestamps.js\n\n# Verify repair\nnode diagnostic-center.js\n```\n\n## 🌐 Network and Performance Issues\n\n### Problem: \"High response times\"\n\n**Symptoms:**\n- API requests taking >5 seconds\n- Job submission timeouts\n- Node registration delays\n\n**Diagnosis:**\n\n1. **Check system resources:**\n```bash\n# CPU and memory usage\ntop -bn1 | head -20\n\n# Disk I/O\niostat -x 1 3\n\n# Disk space\ndf -h\n```\n\n2. **Check database performance:**\n```bash\n# Database size\nls -lh mesh.db\n\n# Count of records\necho \"SELECT \n  (SELECT COUNT(*) FROM jobs) as jobs,\n  (SELECT COUNT(*) FROM nodes) as nodes,\n  (SELECT COUNT(*) FROM ledger) as ledger_entries;\" | sqlite3 mesh.db\n```\n\n**Solutions:**\n- **Large database:** Run cleanup scripts to archive old data\n- **High CPU:** Optimize queries or add indexes\n- **Low memory:** Add swap or upgrade server\n- **Full disk:** Clean up log files and old backups\n\n### Problem: \"Rate limiting blocking legitimate requests\"\n\n**Symptoms:**\n- 429 \"Rate limit exceeded\" errors\n- Nodes unable to register\n- Job submissions rejected\n\n**Diagnosis:**\n```bash\n# Check rate limiting logs\ntail -100 logs/rate-limits.log | grep BLOCKED\n\n# Check current rate limit status\ncurl -v http://localhost:8333/status 2>&1 | grep -i \"rate\\|limit\"\n```\n\n**Solutions:**\n```bash\n# Add IP to whitelist (temporarily)\necho '{\"whitelisted_ips\": [\"YOUR.IP.HERE\"]}' > config/rate-limit-whitelist.json\n\n# Restart server to apply changes\npkill -f server.js && node server.js &\n```\n\n## 🔐 Authentication and API Issues\n\n### Problem: \"API key validation failing\"\n\n**Symptoms:**\n- Valid API keys returning 401 errors\n- Authentication working intermittently\n- \"Invalid API key\" for valid keys\n\n**Diagnosis:**\n\n1. **Test API key format:**\n```bash\n# Check key format (should start with 'sk_')\necho \"$API_KEY\" | head -c 20\n\n# Test key validation\ncurl -H \"X-Api-Key: $API_KEY\" http://localhost:8333/status\n```\n\n2. **Check authentication service:**\n```bash\n# Test if auth service is running\ncurl -k https://127.0.0.1:443/api/auth/verify-key \\\n  -X POST -H \"Content-Type: application/json\" \\\n  -d '{\"key\":\"test-key\"}'\n```\n\n**Solutions:**\n- **Auth service down:** Restart the main site server\n- **Wrong format:** Ensure API key has proper prefix\n- **Network issues:** Check connectivity between mesh and auth servers\n\n## 🔍 Monitoring and Logging\n\n### Setting Up Better Logging\n\n```bash\n# Enable debug logging\nexport DEBUG=ic-mesh:*\n\n# Structured logging with timestamps\nnode server.js 2>&1 | while read line; do \n  echo \"$(date -Iseconds) $line\"; \ndone | tee logs/server-detailed.log\n```\n\n### Monitoring Commands\n\n```bash\n# Watch job queue in real-time\nwatch -n 5 \"echo 'SELECT status, COUNT(*) FROM jobs GROUP BY status;' | sqlite3 mesh.db\"\n\n# Monitor active nodes\nwatch -n 10 \"curl -s http://localhost:8333/nodes | jq 'length'\"\n\n# Track error rate\nwatch -n 30 \"tail -100 mesh.log | grep -i error | wc -l\"\n```\n\n### Health Check Script\n\nCreate a monitoring script:\n\n```bash\n#!/bin/bash\n# health-check.sh\n\n# Colors for output\nRED='\\033[0;31m'\nGREEN='\\033[0;32m'\nYELLOW='\\033[1;33m'\nNC='\\033[0m' # No Color\n\necho \"IC Mesh Health Check - $(date)\"\necho \"================================\"\n\n# Server status\nif curl -s -f http://localhost:8333/status > /dev/null; then\n    echo -e \"✅ Server: ${GREEN}Running${NC}\"\nelse\n    echo -e \"❌ Server: ${RED}Down${NC}\"\n    exit 1\nfi\n\n# Database accessible\nif echo \".databases\" | sqlite3 mesh.db &>/dev/null; then\n    echo -e \"✅ Database: ${GREEN}Accessible${NC}\"\nelse\n    echo -e \"❌ Database: ${RED}Locked/Corrupted${NC}\"\nfi\n\n# Active nodes\nnode_count=$(curl -s http://localhost:8333/nodes | jq 'length' 2>/dev/null || echo 0)\nif [ \"$node_count\" -gt 0 ]; then\n    echo -e \"✅ Active Nodes: ${GREEN}$node_count${NC}\"\nelse\n    echo -e \"⚠️  Active Nodes: ${YELLOW}0${NC}\"\nfi\n\n# Pending jobs\npending=$(echo \"SELECT COUNT(*) FROM jobs WHERE status='pending';\" | sqlite3 mesh.db 2>/dev/null || echo \"N/A\")\necho \"📊 Pending Jobs: $pending\"\n\n# Recent errors\nerrors=$(tail -100 mesh.log 2>/dev/null | grep -i error | wc -l)\nif [ \"$errors\" -gt 10 ]; then\n    echo -e \"⚠️  Recent Errors: ${YELLOW}$errors${NC}\"\nelse\n    echo -e \"✅ Recent Errors: ${GREEN}$errors${NC}\"\nfi\n\necho \"================================\"\necho \"Run 'node diagnostic-center.js' for detailed analysis\"\n```\n\nMake it executable:\n```bash\nchmod +x health-check.sh\n```\n\n## 🆘 Emergency Procedures\n\n### Complete System Reset\n\nIf the system is completely broken:\n\n```bash\n# 1. Stop all processes\npkill -f server.js\npkill -f client.js\n\n# 2. Backup current state\nmkdir -p emergency-backup\ncp mesh.db emergency-backup/mesh-$(date +%Y%m%d-%H%M).db\ncp mesh.log emergency-backup/mesh-$(date +%Y%m%d-%H%M).log\n\n# 3. Reset database (CAUTION: This loses all data)\nrm mesh.db\nnode init-db.js\n\n# 4. Restart server\nnode server.js > mesh.log 2>&1 &\n\n# 5. Verify operation\nsleep 5 && curl http://localhost:8333/status\n```\n\n### Restore from Backup\n\n```bash\n# Find latest backup\nls -lt backups/mesh-*.db | head -1\n\n# Stop server\npkill -f server.js\n\n# Restore database\ncp backups/mesh-YYYYMMDD.db mesh.db\n\n# Restart\nnode server.js > mesh.log 2>&1 &\n```\n\n## 📞 Getting Help\n\n### Information to Gather\n\nWhen reporting issues, include:\n\n```bash\n# System information\nuname -a\nnode --version\nnpm --version\n\n# IC Mesh version\ngrep '\"version\"' package.json\n\n# Recent diagnostic output\nnode diagnostic-center.js > diagnostic-output.txt\n\n# Recent logs (last 100 lines)\ntail -100 mesh.log > recent-logs.txt\n\n# Database stats\necho \".schema\n.databases\nSELECT COUNT(*) as total_jobs FROM jobs;\nSELECT COUNT(*) as total_nodes FROM nodes;\" | sqlite3 mesh.db > db-info.txt\n```\n\n### Common Log Patterns\n\n**Normal operation:**\n- `Node registered:` - New node joined\n- `Job completed:` - Successful job processing\n- `WebSocket connected:` - Node established connection\n\n**Warning signs:**\n- `Rate limit exceeded:` - Too many requests\n- `Database locked:` - Concurrency issues\n- `Handler failed:` - Job processing errors\n- `Quarantined node:` - Poor performing node\n\n**Critical issues:**\n- `ECONNREFUSED` - Service unreachable\n- `SQLITE_CORRUPT` - Database corruption\n- `Out of memory` - Resource exhaustion\n- `Permission denied` - File access issues\n\n## 🎯 Prevention Tips\n\n1. **Regular monitoring:** Set up automated health checks\n2. **Backup strategy:** Daily database backups\n3. **Resource monitoring:** Track CPU, memory, disk usage\n4. **Log rotation:** Prevent log files from growing too large\n5. **Update strategy:** Keep dependencies up to date\n6. **Documentation:** Document any custom configurations\n\nRemember: Most issues are caused by resource constraints, network connectivity, or configuration problems. Start with the basics and work your way up to more complex diagnostics.\n\n---\n\n*This guide is maintained by the IC Mesh community.
